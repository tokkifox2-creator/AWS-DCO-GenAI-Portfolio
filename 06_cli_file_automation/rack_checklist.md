# [교육용 샘플] 랙 현장 점검 체크리스트 (Rack Check-List)

> [!NOTE]
> **[교육용 샘플 데이터 안내]**
> 본 문서는 교육용 가상 시나리오를 바탕으로 작성된 샘플 데이터이며, 실제 운영되는 장비 정보, AWS 내부 절차, 실제 IP 또는 티켓 데이터와 일절 관련이 없습니다.

## 랙 현장 점검 수행 가이드
본 문서는 샘플 로그와 샘플 티켓을 바탕으로 실제 현장에서 랙 점검 시 반드시 확인하고 기록해야 할 항목을 정의한 교육용 체크리스트입니다.

---

### [1단계] 이벤트 및 장비 정보 대조 (Verification)
* [ ] **발생 시간 확인**
  * 스위치 로그 및 티켓에 기록된 최초 에러 발생 시간(`2026-07-07 14:50:00 KST`)과 현장 작업 시간이 일치하거나 인과관계가 맞는지 확인합니다.
* [ ] **샘플 장비명 일치 확인**
  * 점검 대상 랙과 장비명(`SAMPLE_TOR_SW_01`)의 레이블이 실제 현장 랙 구성 정보와 일치하는지 교차 검증합니다. (오접속 방지)
* [ ] **영향 포트 식별**
  * 에러가 보고된 정확한 인터페이스 포트(`SAMPLE_PORT_1/1`)를 랙 다이어그램 상에서 확인합니다.

### [2단계] 상태 분석 및 이력 검토 (Investigation)
* [ ] **이벤트 반복 여부 확인**
  * 해당 포트 혹은 스위치에서 유사한 CRC 에러 증가 및 Link Down 이슈가 과거에 반복적으로 발생한 이력이 있는지 샘플 데이터 히스토리를 확인합니다.
* [ ] **Ticket 상태 확인**
  * 현재 발행된 티켓(`EDU-TICKET-2026-0001`)의 진행 상태가 'Open' 혹은 'In-Progress' 상태인지 확인하고 현장 점검 중임을 업데이트합니다.
* [ ] **원인 단정 금지 (No Assumptions)**
  * 장애 원인을 현장 확인 전 임의로 "케이블 불량"으로만 확정 짓지 않습니다. 트랜시버(SFP) 결합 상태, 광 커넥터 먼지 오염, 스위치 포트 자체의 하드웨어 고장 가능성을 열어두고 다각도로 점검합니다.

### [3단계] 현장 점검 기록 및 문서화 (Reporting)
* [ ] **점검 결과 메모**
  * 포트의 LED 상태(오렌지색 점멸, 소등 등), 광 케이블 굽힘 반경(Bending Radius) 이상 유무 등 실사 내용을 상세히 기록합니다.
* [ ] **보고서 초안 작성 메모**
  * 점검 완료 후 다음 근무자에게 명확한 인수인계를 하거나 에스컬레이션을 위해 점검 결과 초안을 작성하여 인시던트 리포트에 첨부할 준비를 합니다.

---

### ⚠️ 보안 및 규정 준수 (Compliance & Safety)
* **원격 점검 시 외부 대상 점검 명령(예: 실제 외부 IP 대상의 traceroute, nmap 등)을 임의로 수행하지 마십시오.**
* 실무 환경이 아닌 교육용 프로젝트이므로, 실제 장비의 물리적 조작이나 데이터 입력을 수행하지 않습니다.
